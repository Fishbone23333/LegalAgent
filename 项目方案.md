AI Agent 开发指南：应届生法律护航卫士 (LegalShield Agent)
1. 项目愿景 (Project Vision)
本项旨在通过 LangGraph 构建一个具备闭环执行能力的 AI Agent，专门解决大学生在求职（劳动合同）和租房（租赁合同）场景下的法律弱势地位。

Think: 深度解析合同条款，提取关键权责。

Decide: 基于现行法律（劳动法、民法典）判定潜在风险。

Execute: 自动生成法律交涉文案、修订建议或标准的维权存证文档。

2. 技术栈架构 (Tech Stack)
框架: LangGraph (Python) + FastAPI

模型: 推理能力强的 LLM (如 DeepSeek-V3 / Qwen-Max)

前端: Flutter (用于跨端交互演示)

状态管理: 采用 LangGraph 的 TypedDict 维护全局 State

3. LangGraph 数据结构与图逻辑 (Graph Definition)
3.1 状态定义 (State)
Python
from typing import Annotated, List, TypedDict
from langgraph.graph import StateGraph

class AgentState(TypedDict):
    raw_contract: str          # 原始输入文本
    contract_type: str         # 'employment' 或 'housing'
    segments: List[dict]       # 拆解后的条款模块
    risks: List[dict]          # 判定出的风险点 (包含等级、理由、法条依据)
    action_plans: List[str]    # 执行建议
    final_documents: str       # 执行生成的最终文案/修订版合同
3.2 节点逻辑 (Nodes)
节点 A: 提取器 (Thinker - Extractor)
任务: 识别合同类型，并将非结构化文本拆解为：薪资/租金、违约金、解约条件、附加义务。

Prompt 策略: 使用 Few-shot 引导，确保输出严格的 JSON 结构。

节点 B: 风险判定器 (Decider - RiskChecker)
任务: 针对提取出的条款，对比法律基准。

逻辑控制: * 如果违约金 > 月薪/租金的 20% -> 标记为“致命风险”。

如果涉及“自愿放弃社保/公积金” -> 标记为“违法条款”。

输出: 包含 risk_level, legal_basis, recommendation 的列表。

节点 C: 执行生成器 (Executor - DraftGenerator)
任务: 根据风险列表，生成可直接使用的“战斗工具”。

执行产物: 1.  《合同修订建议表》（红线批注版）。
2.  《HR/房东交涉话术模版》（高情商谈判版本）。
3.  《劳动仲裁/投诉证据清单》。

6. 核心 Prompt 引导 (Core Prompting)
系统人设 (System Role)
你是一名拥有 10 年经验、专门保护弱势群体的资深公益律师。你的性格：极其敏锐、言辞专业但通俗易懂、立场坚定。你不仅指出问题（Think），还要给出判定（Decide），并直接产出行动工具（Execute）。

判定逻辑指导 (Decision Logic)
在分析合同时，必须重点检索：

劳动合同：试用期长度、违约金设定、竞业限制范围、加班补偿描述。

租房合同：押金扣除条款、家电维修责任、网络/物业费承担方、提前退租赔偿。
